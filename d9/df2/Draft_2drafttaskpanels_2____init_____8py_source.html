23 """Modules that define classes that handle task panels of the GuiCommands.
24 
25 These classes load `.ui` files that will be used in the task panels
26 of the graphical commands defined in the `draftguitools` submodules.
27 These classes also define the callbacks and interactions of the widgets
28 in the `.ui` files.
29 
30 The task panel interface should be, in the possible, not manually written
31 in Python. Rather a `.ui` file should be built using Qt Designer,
32 and should be placed in the `Resources/ui/` directory so that it is loaded
33 by the respective task panel class.
34 
35 ::
36 
37  class CommandTaskPanel:
38  def __init__(self):
39  ui_file = ":/ui/TaskPanel_OrthoArray.ui"
40  self.form = Gui.PySideUic.loadUi(ui_file)
41 
42 At the moment, most GuiCommands don't have their own `.ui` file,
43 nor task panel class. Instead their task panels are completely defined
44 by the `DraftGui` module.
45 
46 This should be changed in the future because `DraftGui`
47 is a very large module, and creates many different widgets
48 and handles many callbacks, making it difficult to extend and maintain.
49 This module selectively chooses the widgets to show and to hide
50 depending on the command that is activated.
51 
52 Individual task panel classes and `.ui` files are more maintainable
53 because changes can be done to a single tool without affecting the rest,
54 and the module size is kept small.
55 """
