31  // This class implements triangulation of polygons using ear clipping.
32  // The method is O(n^2) for n input points. There are five constructors.
33  // The query type and epsilon parameters are discussed later. In all
34  // cases, the output is
35  //
36  // triangles:
37  // An array of 3*T indices representing T triangles. Each triple
38  // (i0,i1,i2) corresponds to the triangle (P[i0],P[i1],P[i2]),
39  // where P is the 'positions' input. These triangles are all
40  // counterclockwise ordered.
41  //
42  // TriangulateEC(positions,queryType,epsilon,triangles)
43  // positions:
44  // An array of n vertex positions for a simple polygon. The
45  // polygon is (P[0],P[1],...,P[n-1]).
46  //
47  // TriangulateEC(positions,queryType,epsilon,polygon,triangles)
48  // positions:
49  // An array of vertex positions, not necessarily the exact set of
50  // positions for the polygon vertices.
51  // polygon:
52  // An array of n indices into 'positions'. If the array is
53  // (I[0],I[1],...,I[n-1]), the polygon vertices are
54  // (P[I[0]],P[I[1]],...,P[I[n-1]]).
55  //
56  // TriangulateEC(positions,queryType,epsilon,outerPolygon,innerPolygon,
57  // triangles)
58  // positions:
59  // An array of vertex positions, not necessarily the exact set of
60  // positions for the polygon vertices.
61  // outerPolygon:
62  // An array of n indices into 'positions' for the outer polygon.
63  // If the array is (I[0],I[1],...,I[n-1]), the outer polygon
64  // vertices are (P[I[0]],P[I[1]],...,P[I[n-1]]).
65  // innerPolygon:
66  // An array of m indices into 'positions' for the inner polygon.
67  // The inner polygon must be strictly inside the outer polygon.
68  // If the array is (J[0],J[1],...,J[m-1]), the inner polygon
69  // vertices are (P[J[0]],P[J[1]],...,P[J[m-1]]).
70  //
71  // TriangulateEC(positions,queryType,epsilon,outerPolygon,innerPolygons,
72  // triangles)
73  // positions:
74  // An array of vertex positions, not necessarily the exact set of
75  // positions for the polygon vertices.
76  // outerPolygon:
77  // An array of n indices into 'positions' for the outer polygon.
78  // If the array is (I[0],I[1],...,I[n-1]), the outer polygon
79  // vertices are (P[I[0]],P[I[1]],...,P[I[n-1]]).
80  // innerPolygons:
81  // An array of arrays of indices, each index array representing
82  // an inner polygon. The inner polygons must be nonoverlapping and
83  // strictly contained in the outer polygon. If innerPolygons[i]
84  // is the array (J[0],J[1],...,J[m-1]), the inner polygon
85  // vertices are (P[J[0]],P[J[1]],...,P[J[m-1]]).
86  //
87  // TriangulateEC(positions,queryType,epsilon,tree,triangles)
88  // positions:
89  // An array of vertex positions, not necessarily the exact set of
90  // positions for the polygon vertices.
91  // tree:
92  // A hierarchy of nested polygons. The root node corresponds to
93  // the outermost outer polygon. The child nodes of the root
94  // correspond to inner polygons contained by the outer polygon.
95  // Each inner polygon may itself contain outer polygons, and
96  // those outer polygons may themselves contain inner polygons.
97  //
98  // You have a choice of speed versus accuracy. The fastest choice is
99  // Query::QT_INT64, but it gives up a lot of precision, scaling the points
100  // to [0,2^{20}]^3. The choice Query::QT_INTEGER gives up less precision,
101  // scaling the points to [0,2^{24}]^3. The choice Query::QT_RATIONAL uses
102  // exact arithmetic, but is the slowest choice. The choice Query::QT_REAL
103  // uses floating-point arithmetic, but is not robust in all cases. The
104  // choice Query::QT_FILTERED uses floating-point arithmetic to compute
105  // determinants whose signs are of interest. If the floating-point value
106  // is nearly zero, the determinant is recomputed using exact rational
107  // arithmetic in order to correctly classify the sign. The hope is to
108  // have an exact result computed faster than with all rational arithmetic.
109  // The value fEpsilon is used only for the Query::QT_FILTERED case and
110  // should be in [0,1]. If 0, the method defaults to all exact rational
111  // arithmetic. If 1, the method defaults to all floating-point
112  // arithmetic. Generally, if M is the maximum absolute value of a
113  // determinant and if d is the current determinant value computed as a
114  // floating-point quantity, the recalculation with rational arithmetic
115  // occurs when |d| < epsilon*M.
