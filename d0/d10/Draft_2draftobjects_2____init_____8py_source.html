24 """Modules that contain classes that define custom scripted objects.
25 
26 These proxy classes are used to extend one of the core classes
27 defined in C++, for example, `Part::FeaturePython`
28 or `Part::Part2DObjectPython`, thus creating a custom object type
29 tied to a Python class.
30 A new object that uses one of these classes inherits some basic properties
31 from the parent C++ class, and new properties and behavior
32 are added by the Python class.
33 
34 These classes are not normally used by themselves, but are used at creation
35 time by the make functions defined in the `draftmake` package.
36 
37 Once an object is assigned one of these proxy classes, and it is saved
38 in a document, the object will be rebuilt with the same class every time
39 the document is opened. In order to do this, the object module must exist
40 with the same name as when the object was saved.
41 
42 For example, when creating a `Rectangle`, the object uses
43 the `draftobjects.rectangle.Rectangle` class. This class must exist
44 when the document is opened again.
45 
46 This means that, in general, these modules cannot be renamed or moved
47 without risking breaking previously saved files. They can be renamed
48 only if the old class can be migrated to point to a new class,
49 for example, by creating a reference to the new class named the same
50 as the older class.
51 
52 ::
53 
54  old_module.Rectangle = new_module.Rectangle
55 """
