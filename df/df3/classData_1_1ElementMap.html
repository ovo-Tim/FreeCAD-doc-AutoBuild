MappedName Data::ElementMap::setElementName ( const IndexedName &  element,
const MappedName &  name,
long  masterTag,
const ElementIDRefs *  sid = nullptr,

)

Add a sub-element name mapping.

Parameters
elementthe original Type + Index element name
namethe mapped sub-element name. May or may not start with elementMapPrefix().
sidin case you use a hasher to hash the element name, pass in the string id reference using this parameter. You can have more than one string id associated with the same name.
overwriteif true, it will overwrite existing names
Returns
Returns the stored mapped element name.

An element can have multiple mapped names. However, a name can only be mapped to one element

Note: the original proc was in the context of ComplexGeoData, which provided Tag access, now you must pass in long masterTag explicitly.
