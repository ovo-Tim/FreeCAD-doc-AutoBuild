60  // Description:
61  //
62  // This template allows one to define a geometry extension for a given type (uniform interface for one value of type T).
63  //
64  // Warnings:
65  // - The default constructor relies on the default constructor of T for initialisation. Built-in types
66  // so constructed will be uninitialised. Use the specific constructor from a T to initialise it. Note
67  // that the default constructor is required by the type system (see TYPESYSTEM_SOURCE_TEMPLATE_T).
68  //
69  // Default assumptions:
70  // - T can be constructed from T
71  // - T can be assigned to T
72  // - T is convertible to a std::string
73  // - T is serialisable as a string
74  //
75  // template specialisation:
76  //
77  // If the assumptions do not meet for your type, template specialisation allows you to provide specific code,
78  // look for examples (int/string) in GeometryDefaultExtensions.cpp
79  //
80  // Instructions:
81  //
82  // 1. Read the assumptions above and provide template initialisation if needed.
83  // 2. Add an alias to your type under these comments
84  // 3. Add a TYPESYSTEM_SOURCE_TEMPLATE_T in the cpp file to generate class type information
85  // 4. Provide a specialisation of getPyObject to generate a py object of the corresponding type (cpp file)
86  // 5. Provide specialisations if your type does not meet the assumptions above (e.g. for serialisation) (cpp file)
87  // 6. Register your type and corresponding python type in AppPart.cpp
