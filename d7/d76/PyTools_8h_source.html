2 /*************************************************************************
3  * PPEMBED, VERSION 2.0
4  * AN ENHANCED PYTHON EMBEDDED-CALL INTERFACE
5  *
6  * Wraps Python's run-time embedding API functions for easy use.
7  * Most utilities assume the call is qualified by an enclosing module
8  * (namespace). The module can be a file-name reference or a dummy module
9  * created to provide a namespace for file-less strings. These routines
10  * automate debugging, module (re)loading, input/output conversions, etc.
11  *
12  * Python is automatically initialized when the first API call occurs.
13  * Input/output conversions use the standard Python conversion format
14  * codes (described in the C API manual). Errors are flagged as either
15  * a -1 int, or a NULL pointer result. Exported names use a PP_ prefix
16  * to minimize clashes; names in the built-in Python API use Py prefixes
17  * instead (alas, there is no "import" equivalent in C, just "from*").
18  * Also note that the varargs code here may not be portable to certain
19  * C compilers; to do it portably, see the text or file 'vararg.txt'
20  * here, or search for string STDARG in Python's source code files.
21  *
22  * New in this version/edition: names now have a PP_ prefix, files
23  * renamed, compiles to a single .a file, fixed pdb retval bug for
24  * strings, and char* results returned by the "s" convert code now
25  * point to new char arrays which the caller should free() when no
26  * longer needed (this was a potential bug in prior version). Also
27  * added new API interfaces for fetching exception info after errors,
28  * precompiling code strings to byte code, and calling simple objects.
29  *
30  * Also fully supports Python 1.5 module package imports: module names
31  * in this API can take the form "package.package.[...].module", where
32  * Python maps the package names to a nested directories path in your
33  * file system hierarchy; package dirs all contain __init__.py files,
34  * and the leftmost one is in a directory found on PYTHONPATH. This
35  * API's dynamic reload feature also works for modules in packages;
36  * Python stores the full path name in the sys.modules dictionary.
37  *
38  * Caveats: there is no support for advanced things like threading or
39  * restricted execution mode here, but such things may be added with
40  * extra Python API calls external to this API (see the Python/C API
41  * manual for C-level threading calls; see modules rexec and bastion
42  * in the library manual for restricted mode details). For threading,
43  * you may also be able to get by with C threads and distinct Python
44  * namespaces per Python code segments, or Python language threads
45  * started by Python code run from C (see the Python thread module).
46  *
47  * Note that Python can only reload Python modules, not C extensions,
48  * but it's okay to leave the dynamic reload flag on even if you might
49  * access dynamically-loaded C extension modules--in 1.5.2, Python
50  * simply resets C extension modules to their initial attribute state
51  * when reloaded, but doesn't actually reload the C extension file.
52  *************************************************************************/
