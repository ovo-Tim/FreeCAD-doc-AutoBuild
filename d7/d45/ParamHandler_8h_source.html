112 // Helper class to manage handlers of a list of parameters.
113 //
114 // The handlers are stored in a map from ParamKey to shared pointer to a
115 // ParamHandler. The same handler can be registered with multiple keys. When
116 // the registered parameter key is changed, the manager will call the
117 // registered handler function ParamHandler::onChange(). If it returns True,
118 // then the handler will be appended to a queue to be invoked later by a timer
119 // to avoid repetitive processing on change of multiple keys.
120 //
121 // The handler manager is meant to be initiated by some static function, e.g.
122 // DlgSettingsGeneral::attachObserver(). It is intended to be one and only
123 // place of handling changes of the given set of parameters, regardless of
124 // whether the changes are coming from direct editing through parameter editor,
125 // user code changing of parameters, changing preference dialog, or loading a
126 // preference pack.
127 //
